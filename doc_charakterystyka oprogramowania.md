# Charakterystyka oprogramowania

## 1. Nazwa identyfikacyjna

### 1.1. Nazwa skrócona
**TaskPlanner**

### 1.2. Nazwa pełna
**TaskPlanner Desktop Application**

## 2. Sumaryczny opis systemu

### 2.1. Cele projektu
Głównym celem stworzenia aplikacji **TaskPlanner** jest dostarczenie użytkownikom końcowym efektywnego narzędzia wspierającego organizację czasu i zarządzanie zadaniami (ang. *Task Management*). System ma za zadanie:
- Umożliwić cyfrową ewidencję obowiązków z podziałem na kategorie (np. praca, dom).
- Zwiększyć terminowość realizacji zadań dzięki mechanizmom śledzenia dat (ang. *Deadline Tracking*).
- Dostarczyć wizualną informację zwrotną o postępach poprzez interaktywny pulpit sterowniczy (Dashboard).

### 2.2. Opis funkcjonalny
TaskPlanner to nowoczesna aplikacja typu *Desktop*, działająca w środowisku systemu Windows. Wyróżnia się intuicyjnym interfejsem graficznym (GUI) zaprojektowanym w trybie ciemnym (ang. *Dark Mode*), co zapewnia komfort pracy w różnych warunkach oświetleniowych. Dzięki wykorzystaniu lokalnej bazy danych SQLite, aplikacja gwarantuje pełną prywatność danych użytkownika oraz działanie w trybie offline, bez konieczności połączenia z internetem.

## 3. Słownik pojęć i skrótów

W celu zapewnienia jednoznaczności i precyzji w odbiorze dokumentacji, definiuje się następujące terminy:

| Termin / Skrót | Definicja |
|----------------|-----------|
| **Dashboard** | Pulpit sterowniczy; główny ekran aplikacji prezentujący statystyki i podsumowania. |
| **GUI** | Graphical User Interface; graficzny interfejs użytkownika umożliwiający interakcję z programem. |
| **CRUD** | Create, Read, Update, Delete; zestaw podstawowych operacji na danych (tworzenie, odczyt, aktualizacja, usuwanie). |
| **SQLite** | System zarządzania relacyjną bazą danych w modelu bezserwerowym. |
| **Dark Mode** | Tryb ciemny interfejsu, redukujący emisję jasnego światła. |
